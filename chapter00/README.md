# 第0章 はじめに（本コンテンツの前提とスタンス）

## 本コンテンツの目的

本コンテンツは、**CloudFormationを用いたAWS環境構築を学ぶための教育用コンテンツ**です。

そのため、以下のことを目的としています。

- CloudFormationの基本的な考え方を理解する
- CloudFormationテンプレートを読めるようになる
- 簡単なCloudFormationテンプレートを書けるようになる
- 実務でCloudFormationを使う際の最低限の基礎を身につける

一方で下記の内容については本レポジトリの目的とするところではないので、注意してください。

- 商用利用や案件向けテンプレートの提供
- 本番環境で使える完璧なテンプレートの提供
- あらゆるユースケースに対応できる高度な設計手法の習得

## ゴール定義

本コンテンツのゴールは、以下の状態になることです。

> **案件でCloudFormationを使うことになった際に、テンプレートを読み、修正し、最低限の実装ができる状態になること**

具体的には、以下のようなことができるようになることを目指します。

1. **既存のテンプレートを読んで理解できる**
   - 「このテンプレートは何をしているのか」が読み取れる
   - Parameters、Resources、Outputs がそれぞれ何をしているかわかる
   - AWS公式ドキュメントを見ながら、知らないリソースでも調べられる

2. **既存のテンプレートを修正できる**
   - 「この値を変更したい」というときに、どこを変更すればいいかわかる
   - パラメータの追加や削除ができる
   - 既存のリソースに新しいプロパティを追加できる

3. **簡単なテンプレートを一から書ける**
   - VPCやサブネットなど、基本的なリソースを定義できる
   - Parametersを使って環境差分を吸収できる
   - Outputsを使って他のスタックから参照できるようにできる

## 本コンテンツで扱うCloudFormationコードの前提

本コンテンツで扱うCloudFormationテンプレートは、以下の方針で記述されています。

### 可読性と初心者へのわかりやすさを最優先

- **ループ処理や複雑な条件分岐は使わない**
  - `Fn::ForEach`や複雑な`Conditions`は扱いません
  - まずは「普通に書けば、普通に読める」状態を目指します

- **記述の効率性や省略は重視しない**
  - 短く書くことよりも、初心者にとってわかりやすく書くことを優先します
  - 「この書き方のほうが短い」よりも「この書き方のほうが読みやすい」を重視しています

- **高度な設計手法は考え方に触れる程度**
  - ネストスタック、マクロ、カスタムリソースなどは深く扱いません
  - 「こういうものがある」という紹介に留めます

### 「まずは読める・書ける」状態を作る

実務では、複雑なテンプレートや高度な設計手法が必要になる場面もあります。

しかし、まずは「シンプルなテンプレートを読み書きできる」ことが最も重要です。

本コンテンツは、その基礎を固めることに特化しています。

## 学習の進め方

### 1. 各章を順番に読む

本コンテンツは、第1章から順番に読むことを前提としています。

後の章は前の章の内容を前提としているため、できるだけ順番に読み進めてください。

### 2. サンプルテンプレートを実際に動かす

`templates/examples/`ディレクトリには、各章で説明する内容に対応したサンプルテンプレートがあります。

可能であれば、実際にAWS上でデプロイして動作を確認してみてください。

**注意:** AWSの利用料金が発生する可能性があります。不要なリソースは速やかに削除してください。

### 3. 演習問題に取り組む

第9章には演習問題があります。

学習した内容を実際に使ってみることで、理解が深まります。

## 本コンテンツのサポート範囲

本コンテンツは**教育コンテンツとしての提供に限定**されます。

### サポート対象

- CloudFormation の基本的な考え方の説明
- テンプレートの読み書きに関する基礎的な質問
- サンプルコードの理解に関する質問

### サポート対象外

- 実案件・商用システムへの適用可否の判断
- セキュリティ、可用性、コスト最適化に関する個別要件への対応
- 本番環境での利用を前提とした設計レビューや修正対応

本コンテンツは「考え方・書き方を学ぶための教材」であり、**そのまま業務に流用することを前提としたものではありません**。

## 前提知識

本コンテンツを学習する前に、以下の知識があることが望ましいです。

- **AWS の基本的なサービスに関する知識**
  - VPC、Subnet、SecurityGroup、EC2、IAM など
  - 「これらが何をするものか」がわかっていれば十分です

- **YAML の基本的な記法**
  - インデント、マップ、リストなどの基本的な書き方
  - 学習しながらでも問題ありません

- **AWS CLI の基本的な使い方**
  - コマンドを実行できること
  - 学習しながらでも問題ありません
